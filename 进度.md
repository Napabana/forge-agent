## 关键架构决策（记录备查）

1. **内核同步、I/O 异步**：ReAct 主循环同步（清晰可测），WorktreeSession/AgentBus/LLM/子进程
   在边界走 asyncio。`orchestrate_run` 是唯一 async 组合根，用 `to_thread` 桥接。
2. **safe_path 在 executor 层，不在工具层**：`PermissionManager(workspace=)` 强制边界，工具零改动，
   边界由 orchestrator 设（不信任 LLM params）。读写工具都覆盖。
3. **worktree 事务语义不可妥协**：`__aexit__` 始终强制 force 清理；`discard_changes=False` 仅在
   显式 `close` 时保护用户劳动。
4. **Docker 硬隔离 + safe_path 软隔离互补**：Docker 防恶意/失控代码（容器级资源/网络/只读根），
   safe_path 防进程内路径逃逸。两层独立。

## 易踩坑

- `AgentBus.subscribe(topic, maxsize)` 一次只接一个 topic；多 topic 用多个 `messages()` async generator。
- Docker CLI 无 `--nano-cpus` flag（是 API 字段名），用 `--cpus` 换算。
- `event_log` 文件名 `{task_id}_{ts}.jsonl`，含下划线的 task_id 别用 `stem.split("_")[0]` 解析。
- `.gitignore` 有 `*.md`（只留 README.md/USAGE.md）；新增 .md 文档要 `git add -f`。
- API key 只在仓库外 `~/learn-claude-code/.env`，`config/default.yaml` 用 `${VAR}` 占位。
