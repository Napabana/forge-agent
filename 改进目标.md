### 面向软件工程的自主编程智能体引擎项目预期描述

#### 一、 项目总体定位

本项目旨在融合 `forge-agent` 的工程化模块架构与 `s20` 代码的多智能体控制机制（Harness），构建一个面向复杂代码修复与自动化软件工程（SWE）任务的生产级智能体引擎。项目核心理念为“收敛实现路径，优先保障系统级资源隔离与强事务一致性”，避免早期引入过度的分布式中间件，确保整个系统达到“可演示、可测试、逻辑严密”的工业级标准。

---

#### 二、 目标系统架构设计

系统采用混合异步与分层解耦的设计模式，优先保障单机环境下的高可靠性：

* **控制内核层 (Kernel)：** 维持同步的 ReAct 决策主循环以确保执行逻辑的清晰与可测性，同时在 I/O 边界（如 LLM API 请求、子进程调用、Docker 交互）采用 `asyncio` 或 `asyncio.to_thread` 实现异步适配，兼顾决策顺序性与 I/O 吞吐量。
* **状态与任务调度层 (Task Engine)：** 采用单节点高内聚架构，引入 SQLite（配置 WAL 预写日志模式）作为底层状态机，管理有向无环图（DAG）形式的任务依赖。结合 Git Worktree 实现代码层面的事务级隔离。
* **通信与协作层 (Harness & Agent Bus)：** 构建单进程内的异步消息队列（`asyncio.Queue`）作为智能体间通信总线。集成拦截器（Hooks）、权限校验（Permission）、上下文记忆（Memory）以及 MCP（模型上下文协议）。
* **安全沙箱运行时 (Hardened Runtime)：** 深度调用 Docker API，实现容器级别的细粒度隔离与权限收缩。
* **事件审计日志 (EventLog)：** 保留 JSONL 格式的事件日志输出，专用于运行过程回放、端到端演示与系统审计。

---

#### 三、 核心系统工程特性

**1. 声明式工作树事务管理 (Git Worktree + 事务清理)**
系统通过设计 `WorktreeSession` 上下文管理器（包含 `__aenter__` 和 `__aexit__` 协议），实现任务与代码工作区的强制绑定。标准流程为：任务认领（Claim） → 独立 Worktree 创建 → 智能体修改代码 → 自动化测试 → 合并（Merge）或废弃（Discard）。若智能体执行异常或测试未通过，系统将确保底层文件系统强制回滚并清理工作区，保障并发状态下的文件系统一致性。

**2. 高并发安全的状态机持久化 (SQLite WAL)**
废弃传统的纯文本（JSONL）状态机，引入 SQLite 并开启 WAL 模式处理任务图流转。此设计彻底消除了多智能体并发更新状态时的读写冲突与文件锁延迟，提供了完整的 ACID 事务保障，并使复杂任务节点（DAG）的前置依赖计算复杂度大幅降低。

**3. 深度强化的安全沙箱 (Hardened Docker Sandbox)**
摒弃不安全的基础挂载模式，充分利用 Docker 底层能力进行资源与系统调用的严格管控：

* 通过 `--cpus` 与 `--memory` 参数硬性约束计算资源，防止异常代码引发宿主机资源耗尽。
* 默认配置 `--network=none` 阻断非授权网络请求。
* 采用只读根目录（Read-only Root FS）映射，并限定 `tmpfs` 为临时文件写路径。
* 写操作严格限定于当前分配的 Worktree 路径内，配置特定的 Seccomp Profile 拦截危险系统调用。

#### 四、 分阶段实施里程碑 (Milestones)

* **M1：核心事务机制与基础链路 (高优先级)**
* 完成 `WorktreeSession` 上下文管理器封装。
* 设计并初始化 SQLite 数据表结构（`tasks` 与 `task_dependencies`）。
* 集成 JSONL 事件日志，跑通首个基于隔离工作区的单机任务自动修复端到端演示。


* **M2：机制移植与模块化整合**
* 将拦截器、权限控制与上下文提取机制接入现有主循环。
* 实现基于 `asyncio.Queue` 的单进程异步通信总线雏形。


* **M3：运行时沙箱加固**
* 完成 Docker 启动参数的全面接管，实装上述所有安全与资源隔离策略。
* 编写对应的危险命令拦截测试用例。


* **M4：性能调优与横向扩展预留 (按需演进 / Roadmap)**
* **底层异步化：** 对所有高延迟网络 I/O 与文件系统读写进行完整的 `asyncio` 重构。
* **架构升级准备：** 在 README 中规划分布式场景下的 Redis Pub/Sub 升级接口，以及应对极大规模代码库时的 C++17/pybind11 抽象语法树（Tree-sitter）重写方案（仅在产生性能基准测试数据后实施）。

#### 五、 预期简历项目描述呈现

**项目名称：** 自主编程智能体引擎 (Autonomous Coding Agent Engine)
**核心技术：** Python, asyncio, SQLite (WAL), Docker API, Git Worktree, Anthropic/OpenAI 多后端
**项目描述：** 融合模块化 ReAct 框架与多智能体协作协议，面向复杂代码修复与自动化 SWE 任务，独立研发具备强事务一致性与底层资源隔离的智能体运行底座。
**主要工作：**

* **系统状态与一致性保障：** 设计基于 SQLite 的 DAG 任务解析器与 `WorktreeSession` 事务模型；实现任务认领自动隔离代码工作区，并在任务失败或进程崩溃时自动触发文件系统的强制回滚与清理。
* **多智能体核心机制：** 采用混合异步架构，基于单进程内存队列实现异构智能体调度，并集成拦截器（Hooks）、执行权限管控与模型上下文协议（MCP）。
* **内核级安全沙箱：** 强化 Docker 运行时隔离能力，通过底层 API 设置严格的 CPU/内存配额、禁用网络命名空间，并限定写操作路径白名单与只读根目录，拦截未授权的危险系统调用。
* **工程质量保证：** 编写 380+ 单元测试，实现 Issue 提取、逻辑修改至测试通过的端到端自动化演示，保留 JSONL 审计日志支持完整的运行状态回放。
